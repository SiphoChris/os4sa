# OS4SA — Open Source South Africa

**Building for the Public Good.**

The ecosystem for open source software and standards in South Africa. Empowering citizens through interoperable, locally-built infrastructure.

🌐 [os4sa.vercel.app](https://os4sa.vercel.app)

---

## What is OS4SA?

OS4SA is a community-led initiative to foster open source software and standards that serve South African public institutions and citizens. The name encodes the mission: **OS** (open source) **4** (for) **SA** (South Africa).

South Africa has had formal open-source software policy since 2002. Despite this, public universities, municipalities, and government departments remain dominated by expensive, imported, proprietary systems that are ill-fitted to South African requirements. OS4SA exists to close that gap — by building open software, publishing open standards, and giving South African developers the knowledge and community they need to build sustainably in this space.

## Infrastructure Gaps

Critical technical infrastructure that South Africa needs the community to build. These are the missing links — open schemas and API contracts that would benefit every builder in the ecosystem.

| Gap                            | Description                                                                              |
| ------------------------------ | ---------------------------------------------------------------------------------------- |
| **Student Record Interchange** | No portable, open schema for a SA student's academic history across institutions         |
| **NSFAS Data Exchange API**    | Integration is institution-specific and closed; an open contract would benefit every HEI |
| **Municipal Service Requests** | No agreed schema for service delivery requests across 257 municipalities                 |
| **NPO Compliance Format**      | DSD manages 200,000+ NPOs with manual, inconsistent reporting                            |
| **HEMIS Open Schema**          | The data dictionary should be a publicly maintained open schema with validation tooling  |
| **PAIA Request Tracking**      | Departments are legally obligated to process requests but lack any standard tooling      |

If you have built or started work on any of these, open an issue — we want to reference and support your work.

## The Principles

1. Go deep before going wide.
2. Find institutional champions.
3. Design for South African infrastructure.
4. Publish your standards openly.
5. Price for the SMME reality.
6. SA patriotism is a real competitive moat.
7. Compliance is the floor, not the ceiling.
8. Build lean enough to survive procurement.
9. The country needs this. That matters.

## Contributing

Whether you are a developer, a founder, or a civic technologist, there is a place for you in the OS4SA ecosystem.

- **Build something** — tackle one of the infrastructure gaps and open a PR to add it to the ecosystem directory
- **Open an issue** — flag outdated information, propose a new gap, or suggest a project for the directory
- **Share your schemas** — if you have built an open SA-specific data schema, we want to reference it
- **Improve the guide** — corrections, additions, and real-world experience from building in this space are all welcome
